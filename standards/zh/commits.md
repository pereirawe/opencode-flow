# 提交约定

## 格式

```
<类型>: <简短描述> (#<Issue-ID>)
```

## 类型

| 类型 | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | 错误修复 |
| `refactor` | 代码重构 |
| `test` | 测试更改 |
| `docs` | 文档 |
| `chore` | 维护、配置、依赖项 |

## 示例

```
feat: 添加分析端点 (#6)
fix: 在获取前验证 URL 模式 (#3)
refactor: 从处理程序中提取服务层 (#7)
docs: 更新 API 文档 (#12)
```

## 规则

- 使用祈使句现在时
- 第一行保持在 72 个字符以内
- 在括号中引用 Issue 编号
- 正文可选，但对于复杂更改建议使用
