# Directives de Révision de Code

## Principes

- Réviser pour l'exactitude, pas pour le style (le style doit être automatisé)
- Être constructif et spécifique
- Séparer les petites suggestions des problèmes bloquants
- Approuver quand satisfait, demander des changements sinon

## Checklist

- [ ] Le code est correct et gère les cas limites
- [ ] Les tests couvrent adéquatement la modification
- [ ] Pas de régressions de sécurité ou performance
- [ ] Les modifications d'API sont documentées
- [ ] Les chemins d'erreur sont traités
- [ ] Pas de code mort ou commenté
- [ ] Les conventions du projet sont respectées

## Flux de Révision

1. Committer effectue la révision initiale
2. Publish Requester assigne les Senior Reviewers par domaine
3. Les Senior Reviewers révisent en parallèle
4. Tous les réviseurs doivent approuver avant le merge
