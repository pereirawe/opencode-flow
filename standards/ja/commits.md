# コミット規約

## 形式

```
<タイプ>: <短い説明> (#<Issue-ID>)
```

## タイプ

| タイプ | 使用目的 |
|--------|----------|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `refactor` | コード再構築 |
| `test` | テスト変更 |
| `docs` | ドキュメント |
| `chore` | メンテナンス、設定、依存関係 |

## 例

```
feat: アナリティクスエンドポイントを追加 (#6)
fix: フェッチ前にURLスキーマを検証 (#3)
refactor: ハンドラからサービス層を抽出 (#7)
docs: APIドキュメントを更新 (#12)
```

## ルール

- 命令形の現在形を使用
- 最初の行は72文字未満に保つ
- Issue番号を括弧内に参照
- 本文はオプションだが、複雑な変更には推奨
