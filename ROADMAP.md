# Roadmap

## Short Term (next weeks)

- [x] Version bump automation (`ocf:bump-version`)
- [x] Prioritization file moved to project root
- [ ] **Multi-tool support**: generate config for Cursor, Windsurf, Claude Code, Continue.dev
- [ ] **Badges and social preview** for GitHub repo
- [ ] **Starter template repo** — demo project using the flow end-to-end

## Medium Term (next months)

- [ ] **`npx opencode-flow init`** — one-command setup for any project
- [ ] **VS Code extension** — pipeline status, issue browser, agent launcher
- [ ] **Pipeline dashboard** — local web UI showing issue lifecycle
- [ ] **CI/CD integration** — GitHub Action that validates pipeline gates on PR
- [ ] **Plugin system** — custom agents per project domain
- [ ] **More locale translations** (fr, de, ja, zh already have standards)

## Long Term (vision)

- [ ] **OpenCode Flow as a service** — hosted pipeline with Slack/email notifications
- [ ] **AI-generated acceptance criteria** from business rules
- [ ] **Automatic test generation** from acceptance criteria
- [ ] **Cross-project dependency tracking** — link issues across repos
- [ ] **Community agent marketplace** — share and discover custom agents

## How to Influence the Roadmap

Open a [Discussion](https://github.com/pereirawe/opencode-flow/discussions) or
upvote existing ones. Priority is driven by community demand.
